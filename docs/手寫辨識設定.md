# 國語手寫辨識（全開源、不需 API Key）



辨識順序：



1. **筆畫辨識** — [hanzilookup-js](https://www.npmjs.com/package/hanzilookup-js) + [HanziLookup](https://github.com/gugray/HanziLookupJS) 字庫（Make Me a Hanzi 資料，開源）  

2. **PaddleOCR.js** — [PP-OCRv5](https://github.com/PaddlePaddle/PaddleOCR) mobile 模型（開源，手寫優於 Tesseract）  

3. **同音字四選一** — 從你的試算表題庫依注音挑字（不需網路 API）  



全部失敗時會出現 **4 個同音字** 讓孩子點，不必金鑰、不必雲端辨識帳號。



---



## 筆畫辨識（建議、單字最準）



- 第一次會從 GitHub **下載字庫**（約數 MB，快取後不用重下）  

- 請**一筆一筆寫**、字寫大一點（單字、雙字較適合）  

- 關閉：`js/config.site.js` → `HANZI_STROKE_ENABLED: false`  



## PaddleOCR 圖像辨識



- 從 jsDelivr 載入 `@paddleocr/paddleocr-js`，**首次約 40～60 MB**（含模型，瀏覽器會快取）  

- 進入國語測驗時會在背景預載；送出第一題若尚未載完會多等一會兒  

- 關閉：`OCR_ENABLED: false`（可只靠筆畫 + 四選一）  

- `OCR_USE_WHITELIST: true` 時只保留標準答案裡出現的字（減少辨成別字）  



## 同音四選一



- `HOMOPHONE_PICKER: true`（預設開啟）  

- `HOMOPHONE_ONLY_SIMILAR: true`（預設）：只有**同音易混**才四選一；明顯寫錯改「請再寫一次」  

- 辨識完全失敗時仍會四選一（依注音點選）  

- 題庫愈多，同音選項愈豐富  



---



## 為什麼不用 Google Vision？



雲端 OCR 要 API Key 與帳單設定。本 App 只用開源方案；**四選一**仍是低年級最穩的備援。



## 技術說明



- `index.html` 的 **import map** 指向 PaddleOCR.js CDN  

- 模型由 Paddle 官方 CDN 下載（`PP-OCRv5_mobile_det` / `rec`）  

- ONNX Runtime Web WASM 來自 jsDelivr  



若網路很慢，可讓孩子先寫幾題，等快取完成後會順很多。
